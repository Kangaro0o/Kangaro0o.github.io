本地存在的项目如何同步到 GitHub

2022-06-29 · 2 min read
GitHub

前言

相信不好同学都遇到要把本地已创建好的项目或者一个完善项目同步到 GitHub 上的问题。今天简单写个教程记录下。

操作指南

1. 初始化本地项目

在要同步到 GitHub 的项目目录下使用

git init

初始化本地项目，执行此命令后，项目目录下会生成一个 .git 目录。

2. 本地文件添加到暂存区

在项目目录下执行如下命令

git add .

不要忘记后面的 .，. 表示添加该目录下所有文件，执行此命令后，项目目录下所有文件都会添加到 git 暂存暂存区中。

3. 本地提交 Git 暂存文件

接下来使用如下命令提交暂存区文件：

git commit -m "commit log"

-m后面为本次提交说明

4. 关联远程仓库

使用如下命令

git remote add origin git@github.com:Kangaro0o/leetcode-go.git

关联到 GitHub 的远程仓库 git@github.com:Kangaro0o/leetcode-go.git，你需要把这个替换成自己的仓库地址。

4.1 报错问题

如果执行到第4步报错：

fatal: remote origin already exists

可以尝试如下解决办法：

git remote rm origin
git remote add origin git@github.com:Kangaro0o/leetcode-go.git

先删除已存在的关联仓库，再重新进行关联操作。

5. 拉取远程仓库数据

项目目录下执行

git pull --rebase origin master

拉取远程仓库文件，如果远程仓库不为空，则必须要做这一步，不然后面的提交会失败。

6. 把本地仓库数据推送到远程

项目目录下执行

git push -u origin master

把本地仓库文件推送到 GitHub 远程仓库。

如果没有报错就可以在 GitHub 上看到自己的项目了。
